Итог

Arduino Uno — «скачал и работает» у всех. ESP32 — только при установке в ASCII-путь.

  • Ядро подтверждено: приложение запускается, встроенные серверы поднимаются, тулчейн собирает программы и для Uno, и для ESP32 полностью офлайн.
  • Arduino Uno устойчив к кириллице — компилируется даже из установки в папку с кириллическим именем и при кириллическом имени Windows-аккаунта. Для твоих завтрашних планов с Uno — зелёный свет.
  • ESP32 ломается на кириллице в пути установки — Rust-инструмент сборки ESP32 не умеет не-ASCII пути. Это единственный барьер для «у всех работает», и он решается выбором ASCII-папки установки.

Критичный риск

Кириллица в пути установки ломает ESP32 (не Uno)

⚠ Пользователь с кириллицей в пути установки или имени аккаунта не сможет собрать ESP32

Тулчейн ESP32 содержит Rust-инструмент, который падает на не-ASCII путях (Failed to get path name. Error code: 3). Ломается именно путь к папке установки (там лежит tools/Arduino) — build-папка и имя аккаунта роли не играют. Arduino Uno этим не затронут.

Как обеспечить «работает у всех» (без правки логики приложения):

1. Установка для всех пользователей → C:\Program Files\OpenBlockDesktop — путь всегда ASCII, ESP32 работает независимо от кириллического имени аккаунта.
2. Либо дефолт установщика перевести на фиксированный ASCII-путь (например C:\OpenBlock) и/или запретить выбор кириллической папки — это настройка NSIS, не код.
3. Правильное решение «навсегда» — CI-сборка + мелкий патч openblock-link, копирующий тулчейн в ASCII-temp (в бэклоге).

Раздел B · Ручные тест-кейсы (завтра, с платой)

Полный путь «чистый пользователь → мигающий светодиод»

Разверни любой кейс. Проходи по порядку; каждый — это «Предусловие → Шаги → Ожидаемо». Uno проверяем физически, ESP32 — с оговоркой про ASCII-путь.

TC-01Установка приложения›
Предусл.Скачан OpenBlock-Desktop_v2.6.3_win_x64.exe. Диск с ≥ 8 ГБ свободного.
Шаги
  1. Двойной клик по .exe.
  2. На запрос UAC — «Да».
  3. Указать ASCII-путь установки (латиница, напр. C:\Program Files\OpenBlockDesktop). Кириллицу в пути — не использовать.
  4. Дождаться распаковки (~6.5 ГБ, пара минут), «Готово».
ОжидаемоУстановка завершается, создаётся ярлык, приложение запускается.
TC-02Первый запуск и загрузка интерфейса›
Шаги
  1. Открыть приложение (или оно откроется само после установки).
  2. Дождаться загрузки.
ОжидаемоОкно открывается без ошибок; видно сцену, палитру блоков и панель устройства. Внутренние серверы (20111/20112) поднимаются автоматически.
TC-03Выбор устройства Arduino Uno›
Шаги
  1. Открыть выбор устройства (иконка платы / «Add device»).
  2. Выбрать Arduino Uno.
ОжидаемоUno появляется в списке и выбирается; открывается экран подключения; палитра пополняется блоками пинов/Serial.
TC-04Подключение платы по USB›
Предусл.Кабель с данными (не «только зарядка»).
Шаги
  1. Воткнуть Uno в USB.
  2. В приложении выбрать появившийся COM* порт.
  3. Нажать «Connect».
ОжидаемоПорт появляется в списке, статус меняется на «подключено».
Если нет портаСменить кабель; поставить драйвер CP2102/CH340 из папки drivers установки.
TC-05Собрать программу из блоков (мигание)›
Шаги
  1. Взять стартовый блок (программа/старт).
  2. Внутри «forever»: «set pin 13 HIGH» → «wait 0.5 s» → «set pin 13 LOW» → «wait 0.5 s».
ОжидаемоБлоки собираются в стек без ошибок соединения.
TC-06Проверка сгенерированного кода›
Шаги
  1. Переключиться в вид кода (Arduino C++).
ОжидаемоВиден корректный скетч: pinMode(13,OUTPUT), digitalWrite(13,HIGH/LOW), delay(500) — соответствует блокам.
TC-07Компиляция›
Шаги
  1. Нажать «Upload» (или отдельную компиляцию).
ОжидаемоПрогресс сборки → «compile success» без ошибок. Проверено автотестом AT-03 — компилятор Uno работает.
TC-08Прошивка на плату›
Шаги
  1. Дождаться, пока после компиляции пойдёт заливка (avrdude для Uno).
ОжидаемоПрогресс записи → «upload complete». RX/TX-светодиоды платы мигают во время заливки.
Если виситПроверить, что порт не занят Serial-монитором; переподключить плату.
TC-09Проверка на железе (главный кейс)›
ОжидаемоВстроенный светодиод Uno (пин 13, метка «L») мигает раз в 0.5 c. Это и есть «оно реально работает на плате».
ВариацияПоменять задержку на 0.1 c, перезалить — частота мигания меняется.
TC-10Serial-монитор›
Шаги
  1. Добавить блок печати в Serial в цикл, перезалить.
  2. Открыть Serial-монитор на 9600.
ОжидаемоВ мониторе идёт поток строк в такт программе.
TC-11ESP32 — тот же путь ESP32›
Предусл.Приложение установлено в ASCII-путь (см. критичный риск). Плата ESP32 + драйвер CP2102/CH340.
Шаги
  1. Выбрать устройство ESP32, подключить COM, собрать мигание на пине 2, залить.
ОжидаемоКомпиляция и прошивка проходят (AT-04 подтверждает компилятор), светодиод на GPIO2 мигает.
Негатив-кейсЕсли установлено в кириллическую папку — компиляция ESP32 упадёт с «Failed to get path name». Это ожидаемое поведение бага AT-11.
TC-12Работа офлайн›
Шаги
  1. Отключить интернет.
  2. Повторить компиляцию + прошивку.
ОжидаемоВсё работает — тулчейн, ядра и драйверы вложены в установку, сеть не нужна.
TC-13Драйвер USB-UART (если порт не виден)›
Шаги
  1. Определить чип платы (CP2102 = Silicon Labs, CH340).
  2. Поставить драйвер из папки drivers установки.
  3. Переподключить плату.
ОжидаемоПосле установки драйвера появляется COM* порт.

Раздел C · Виртуальная проверка (без платы)

Симуляция Arduino Uno в Wokwi на нашей же прошивке

  • Проект готов: E:\wokwi-uno (wokwi.toml + diagram.json), прошивка fw\sketch.ino.hex скомпилирована нашим тулчейном (AT-06 — HEX валиден).
  • В открытом VS Code: F1 → Wokwi: Start Simulator.
  • Первый раз — «Request a new license» (бесплатно, разовый вход в аккаунт Wokwi).
  • Ожидаемо: на виртуальной Uno мигает зелёный светодиод (D13), в Serial-мониторе идёт LED ON / LED OFF.
